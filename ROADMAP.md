# 🗺️ Ecosystème 
Évolution du projet depuis la version 1.0.0 jusqu’à la version 4.1.0.  
Objectif : construire progressivement un écosystème complet d’analyse, de simulation, de réglage et d’optimisation moteur.

---

## 🟦 v1.0.0 — Création du projet
### Objectifs
- Initialiser la base du projet Racing50cc.
- Définir les premiers modules techniques.
- Structurer les dossiers principaux.

### Livrables
- Structure initiale du repo.
- README.md de base.
- Dossiers : `/engine`, `/transmission`, `/docs`.
- Premiers tests manuels.

---

## 🟩 v1.1.0 — Premiers modules techniques
### Objectifs
- Ajouter les premiers outils d’analyse et de calcul.
- Préparer les futurs modules de tuning.

### Livrables
- Module transmission (calcul vitesse théorique).
- Module carburateur (gicleurs).
- Module thermique (T°C culasse).
- Premiers scripts d’analyse.

---

## 🟫 v1.5.0 — Structure & Organisation
### Objectifs
- Réorganiser le projet pour accueillir les futures versions.
- Structurer les tests et les performances.

### Livrables
- Dossiers : `/performance`, `/simulations`, `/tests_dynamiques`.
- Templates de tests.
- Documentation interne.
- Harmonisation CSS/JS.

---

## 🟪 v2.0.0 — Tuning Avancé
### Objectifs
- Introduire les réglages moteur avancés.
- Préparer les simulateurs.

### Livrables

- Analyse AFR.
- Analyse thermique.

## 🟨 v2.5.0 — Simulateurs
### Objectifs

### Livrables

## 🟦 v3.0.0 — Dashboard & Analyse Avancée
### Objectifs
- Centraliser l’analyse dans un dashboard moderne.
- Comparer les runs et exploiter les données brutes.

### Livrables
- Dashboard performance (RPM, vitesse, AFR, T°C).

- Analyseur de données brutes.

- Structure tests dynamiques.
- Documentation technique.

---

## 🟧 v3.5.0 — Track Pack (Préparation)
### Objectifs
- Préparer les réglages piste et les conditions réelles.
- Étendre les capacités d’analyse.

### Livrables

## 🟩 v4.0.0 — Performance+ / Simulation+ / Optimisation+
### Objectifs

### Livrables

## 🟩 v4.1.0 — Tests_extras Expansion Pack
### Objectifs
- Étendre les tests avancés hors protocole standard.
- Structurer l’analyse, la calibration et l’export.

### Livrables
- Création du dossier **tests_extras/**.
- Sous-dossiers :
  - **analysis/** (analyse approfondie)
  - **calibration/** (procédures moteur & capteurs)
  - **export/** (formats d’export optimisés)

  - README.md
  - INFORMATION.rst (pour chaque sous-dossier)
- Fichiers internes :
  - scripts d’analyse
  - templates de rapports
  - données d’exemple
  - procédures de calibration
  - formats d’export
